## Package Overview

ggpubr is an R package that wraps ggplot2 to produce publication-ready plots with a simplified API. It targets researchers without advanced R programming skills. Current dev version: 0.6.2.999.

## Development Commands

```r

devtools::load_all()

devtools::test()

# Run a single test file
```

## Architecture

### Plot Function Pattern
Every high-level plot function (ggscatter, ggboxplot, gghistogram, etc.) follows the same structure:
1. Accept `data`, `x`, `y` (and plot-specific params)
2. Build geom layers via `geom_exec()` — which separates column-name aesthetics from static values
3. Apply global formatting through `ggpar()` (themes, fonts, legends, colors, scales)
4. Return a standard ggplot object

Parameters are inherited across functions using `@inheritParams ggpar` and `@inheritParams ggboxplot` in roxygen2 comments. Shared parameter docs live in `R/shared_docs.R`.

### Stat/Geom Extensions
Custom ggplot2 stat layers (stat_cor, stat_compare_means, stat_anova_test, geom_pwc, geom_bracket, etc.) follow the standard ggproto pattern: a user-facing function that calls `ggplot2::layer()` paired with a `ggplot2::ggproto()` Stat/Geom object with `compute_group` or `compute_panel` methods.

### Key Internal Functions
- **`geom_exec()`** (`R/geom_exec.R`): Central dispatcher that separates aes mappings from static options using an `allowed_options` whitelist, then calls the geom function
- **`create_aes()`** (`R/create_aes.R`): Builds `aes()` mappings, handling the `ggpubr.parse_aes` option for non-standard column names (spaces, hyphens)
- **`ggpar()`** (`R/ggpar.R`): Post-processing function applied to all plots for consistent theming, axis labels, legend, color palette, etc.
- **`.barplot_params()`, `.boxplot_params()`, etc.** (`R/utilities.R`): Extract and default geom-specific parameters from `...`

### Global Options
- `ggpubr.parse_aes` (default TRUE): Controls whether column names are parsed/unquoted. Set to FALSE for column names containing special characters like hyphens.
- `ggpubr.null_device`: Graphics null device for `as_ggplot()`/`ggarrange()` rendering (default: `cowplot::pdf_null_device`).

### Key Dependencies
- **rstatix**: Provides the underlying statistical tests used by `compare_means()` and stat layers
- **cowplot**: Used for plot arrangement (`ggarrange`) and null device management
- **ggsignif**: Provides `geom_signif` (re-exported) for significance brackets

### Testing
Tests use testthat in `tests/testthat/`. The standard pattern builds a ggplot object, extracts layer data via `ggplot2::ggplot_build(p)$data[[1]]`, and asserts on the resulting data frame. Tests use base R datasets (mtcars, ToothGrowth) for reproducibility.

### File Organization
- `R/utilities.R`, `R/utilities_base.R`, `R/utilities_color.R`, `R/utilities_label.R`: Internal helpers
- `R/gg*.R`: High-level plot functions
- `R/stat_*.R`, `R/geom_*.R`: ggplot2 stat/geom extensions
- `R/theme_pubr.R`: Publication-ready themes
- NAMESPACE is auto-generated by roxygen2 — do not edit by hand

### Collation Order
Source file load order is explicitly specified in DESCRIPTION's `Collate:` field. Utility files load first (utilities_color.R → utilities_base.R → utilities.R), then feature files. When adding new files, update the Collate field or use `@include` roxygen tags.
